A/B Testing a User Interface

Overview

I was tasked with determining how a design choice could impact User Interaction by running A/B testing on two different web designs. I will do this by gathering data on user interaction and analyzing the results using t-tests, an inferential statistic used to evaluate significant differences between two groups' means and their relationship.

The first UI I was given and with the second one I redesigned the first one. The website was formatted for someone who intended to make a doctor's appointment and the site gives information on which days certain doctors are available at a certain place. Some of the changes I made were changing the color of buttons to make less confusion around which button was “scheduling an appointment” and which button went to the “see appointment” page. I also changed the format of the buttons and created a line between each appointment to distinguish between them all.

\

Data Colletion

This assignment was done in a class, and so the data I collected was from a lab we all participated in. We first all ‘tested’ the A version of the interface. We did this by being tasked with scheduling an appointment on a specific day, with a certain doctor. After we all did this, we were able to make changes to the website ended up being version B. Then, went around and tested each other's designs. It was the same people who tested version A that tested version B.
